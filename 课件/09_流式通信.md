# 第 09 课：流式通信 —— gRPC 的最大杀器

> **目标**：掌握三种流式 RPC —— 服务端流、客户端流、双向流  
> **价值**：这是 REST 最难做到的事，却是 gRPC 的原生能力

---

## 零、回顾：四种 RPC 类型

| 类型 | Proto 定义 | 客户端 | 服务端 | 典型场景 |
|------|-----------|--------|--------|----------|
| 一元 RPC | `rpc X(A) returns (B)` | 发一个，收一个 | 收一个，回一个 | 普通 CRUD |
| **服务端流** | `rpc X(A) returns (stream B)` | 发一个，收多个 | 收一个，发多个 | 列表、推送 |
| **客户端流** | `rpc X(stream A) returns (B)` | 发多个，收一个 | 收多个，回一个 | 批量上传 |
| **双向流** | `rpc X(stream A) returns (stream B)` | 发多个，收多个 | 收多个，发多个 | 聊天、协作 |

> 前面课已经学了一元 RPC。这一课学剩下的三种。

## 一、服务端流式 —— 服务端推送多条数据

### 场景

> 查询所有用户 → 服务端逐个推送，客户端像读文件一样迭代。

### 1.1 新增 proto 定义

在 `user.proto` 中添加：

```protobuf
service UserService {
    // ... 之前的一元 RPC ...

    // 服务端流式：客户端发一个请求，服务端返回多条数据
    rpc ListUsers (ListUsersRequest) returns (stream UserResponse);
}

message ListUsersRequest {
    int32 page_size = 1;   // 可选：每页多少条
}
```

> 重新编译：`python -m grpc_tools.protoc -I. --python_out=. --grpc_python_out=. user.proto`

### 1.2 服务端实现

```python
def ListUsers(self, request, context):
    """
    服务端流式：用 yield 逐个返回数据
    
    客户端会收到一个迭代器，for 循环逐个读取
    """
    print(f"[ListUsers] 开始推送，共 {len(self.db)} 个用户")

    count = 0
    for user_id, user in self.db.items():
        if request.page_size > 0 and count >= request.page_size:
            break
        yield user          # ← 不是 return，是 yield！
        count += 1
        import time
        time.sleep(0.5)     # 模拟间隔推送（实际项目不需要 sleep）

    print(f"[ListUsers] 推送完成，已推送 {count} 条")
```

### 1.3 客户端调用

```python
print("[LIST] 获取用户列表（服务端流式推送）\n")

# 调用服务端流式方法，返回一个迭代器
responses = stub.ListUsers(
    user_pb2.ListUsersRequest(page_size=0),  # 0 = 不限制
    timeout=10.0
)

# 像遍历列表一样逐个接收！
for i, user in enumerate(responses, 1):
    print(f"   [{i}] ID={user.id}, Name={user.name}, Email={user.email}")

print(f"\n[OK] 共收到 {i} 条数据")
```

### 1.4 运行效果

```
[LIST] 获取用户列表（服务端流式推送）

   [1] ID=1, Name=Alice, Email=alice@example.com
   [2] ID=2, Name=Bob, Email=bob@example.com
   [3] ID=3, Name=Charlie, Email=charlie@example.com

[OK] 共收到 3 条数据
```

## 二、客户端流式 —— 客户端批量上传

### 场景

> 批量创建用户 → 客户端逐个发送，服务端收齐后汇总返回。

### 2.1 新增 proto 定义

```protobuf
// 客户端流式：客户端发多个请求，服务端返回一个汇总
rpc BatchCreateUsers (stream CreateUserRequest) returns (BatchCreateResponse);

message BatchCreateResponse {
    int32 success_count = 1;
    int32 fail_count = 2;
    repeated int32 created_ids = 3;   // repeated = 数组
    repeated string errors = 4;
}
```

### 2.2 服务端实现

```python
def BatchCreateUsers(self, request_iterator, context):
    """
    客户端流式：从 request_iterator 逐个读取请求，最后返回一个汇总
    
    request_iterator 是一个迭代器，每次迭代拿到客户端发来的一个请求
    """
    created_ids = []
    errors = []

    for request in request_iterator:     # ← 逐个读取客户端发来的请求
        print(f"[BatchCreate] 收到: {request.name}")

        # 参数校验
        if not request.name.strip():
            errors.append(f"创建失败: 名称为空")
            continue

        # 创建用户
        user = user_pb2.UserResponse(
            id=self.next_id,
            name=request.name,
            email=request.email
        )
        self.db[self.next_id] = user
        created_ids.append(self.next_id)
        self.next_id += 1

    # 汇总返回
    return user_pb2.BatchCreateResponse(
        success_count=len(created_ids),
        fail_count=len(errors),
        created_ids=created_ids,
        errors=errors
    )
```

### 2.3 客户端调用

```python
print("[BATCH] 批量创建用户（客户端流式上传）\n")

# 方式1：传一个生成器
def user_generator():
    users = [
        ("张三", "zhangsan@example.com"),
        ("李四", "lisi@example.com"),
        ("", "bad@example.com"),           # 这个会失败（空名称）
        ("王五", "wangwu@example.com"),
    ]
    for name, email in users:
        print(f"   → 发送: {name or '(空名称)'}")
        yield user_pb2.CreateUserRequest(name=name, email=email)

# 发起客户端流式调用
resp = stub.BatchCreateUsers(
    user_generator(),
    timeout=10.0
)

print(f"\n[OK] 批量创建结果:")
print(f"   成功: {resp.success_count} 个")
print(f"   失败: {resp.fail_count} 个")
print(f"   新 ID: {list(resp.created_ids)}")
if resp.errors:
    for e in resp.errors:
        print(f"   错误: {e}")
```

### 2.4 运行效果

```
[BATCH] 批量创建用户（客户端流式上传）

   → 发送: 张三
   → 发送: 李四
   → 发送: (空名称)
   → 发送: 王五

[OK] 批量创建结果:
   成功: 3 个
   失败: 1 个
   新 ID: [4, 5, 6]
   错误: 创建失败: 名称为空
```

## 三、双向流式 —— 双方同时读写

### 场景

> 实现一个简单的聊天室 → 客户端持续发送消息，服务端持续回复。

### 3.1 proto 定义

```protobuf
// 双向流式：双方都可以持续发送和接收
rpc Chat (stream ChatMessage) returns (stream ChatMessage);

message ChatMessage {
    string user = 1;
    string text = 2;
    string timestamp = 3;
}
```

### 3.2 服务端实现

```python
import time as _time

def Chat(self, request_iterator, context):
    """
    双向流：从 request_iterator 读消息，同时 yield 回复
    
    每收到一条客户端消息，就回复一条确认
    """
    for msg in request_iterator:
        print(f"[Chat] {msg.user}: {msg.text}")

        # 回复确认消息
        response = user_pb2.ChatMessage(
            user="Server",
            text=f"已收到: {msg.text}",
            timestamp=_time.strftime("%H:%M:%S")
        )
        yield response
```

### 3.3 客户端实现

```python
import threading

def chat():
    channel = grpc.insecure_channel('localhost:50051')
    stub = user_pb2_grpc.UserServiceStub(channel)

    # 双向流调用返回一个特殊的对象，可以同时 send 和 receive
    call = stub.Chat(timeout=30.0)

    def receive_messages():
        """接收线程：持续读取服务端的回复"""
        for msg in call:
            print(f"   [MSG] [{msg.timestamp}] {msg.user}: {msg.text}")

    # 启动接收线程
    receiver = threading.Thread(target=receive_messages, daemon=True)
    receiver.start()

    # 主线程：发送消息
    messages = [
        ("Alice", "你好！"),
        ("Alice", "今天天气不错"),
        ("Alice", "再见！"),
    ]
    for user, text in messages:
        msg = user_pb2.ChatMessage(
            user=user,
            text=text,
            timestamp=_time.strftime("%H:%M:%S")
        )
        print(f"   [SEND] {user}: {text}")
        call.write(msg)
        _time.sleep(1)

    # 发送完毕，通知服务端
    call.done_writing()

    # 等接收线程处理完
    receiver.join(timeout=5)
    channel.close()

chat()
```

### 3.4 运行效果

```
   [SEND] Alice: 你好！
   [MSG] [14:30:01] Server: 已收到: 你好！
   [SEND] Alice: 今天天气不错
   [MSG] [14:30:02] Server: 已收到: 今天天气不错
   [SEND] Alice: 再见！
   [MSG] [14:30:03] Server: 已收到: 再见！
```

## 四、三种流式速查卡

| | 客户端代码 | 服务端代码 | 数据流向 |
|------|-----------|-----------|:---:|
| **一元** | `r = stub.X(req)` | `return resp` | Client → Server |
| **服务端流** | `for r in stub.X(req):` | `yield resp` × N | Server ⇢⇢ Client |
| **客户端流** | `r = stub.X(gen())` | `for req in iterator:` → `return resp` | Client ⇢⇢ Server |
| **双向流** | `call.write()` + `for r in call:` | `for req in iterator:` + `yield` | Client ⇄⇄ Server |

---

## 五、真实应用场景

| 流类型 | 常见场景 |
|--------|----------|
| 服务端流 | 大数据列表分页推送、实时通知推送、日志监控流、订阅推送 |
| 客户端流 | 批量上传文件、批量导入数据、客户端遥测数据上报 |
| 双向流 | 聊天系统、实时协作编辑、Ride-hailing 位置更新、IoT 双向遥测 |

## 六、本课小结

```
一元 RPC 你已掌握（第04-05课）。
现在你学会了全部四种 RPC：

   一元：      stub.X(req)     →   return resp
   服务端流：   for r in stub.X  →   yield resp
   客户端流：   stub.X(gen())    →   for req in iter: return
   双向流：    write() + read()  →   yield while read()

   记住：REST 做流式需要 WebSocket/SSE，gRPC 原生支持。
   这就是 gRPC 在微服务通信中最大的优势。
```
