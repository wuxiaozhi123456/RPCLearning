# 第 08 课：拦截器（Interceptor） —— gRPC 的 Middleware

> **目标**：学会编写服务端和客户端的拦截器，实现日志记录、认证注入等横切关注点  
> **对照**：gRPC Interceptor ≈ FastAPI Middleware / Spring Interceptor

---

## 一、回忆：FastAPI 的 Middleware

```python
# FastAPI Middleware —— 每个请求都会经过
@app.middleware("http")
async def log_requests(request, call_next):
    print(f"[收到] {request.method} {request.url}")
    start = time.time()
    response = await call_next(request)
    print(f"[完成] 耗时: {time.time() - start:.2f}s")
    return response
```

> gRPC 的拦截器做的是同样的事 —— 在每个 RPC 调用前后插入自定义逻辑。

---

## 二、gRPC 拦截器类型

| 类型 | 位置 | 作用 |
|------|------|------|
| 服务端拦截器 | `ServerInterceptor` | 在服务端收到请求时执行 |
| 客户端拦截器 | `UnaryUnaryClientInterceptor` 等 | 在客户端发起请求时执行 |

> 这里先学最常用的两种：服务端通用拦截器 + 客户端一元拦截器。

---

## 三、服务端日志拦截器

### 代码

```python
import grpc
import time
from grpc import ServerInterceptor

class LoggingInterceptor(grpc.ServerInterceptor):
    """
    服务端日志拦截器：
    记录每个 RPC 请求的方法名、处理耗时
    """

    def intercept_service(self, continuation, handler_call_details):
        """
        continuation: 调用链中的下一个拦截器（或最终的处理函数）
        handler_call_details: 请求的元信息（包含方法名等）
        """
        method = handler_call_details.method
        print(f"[gRPC Server] → 收到请求: {method}")

        start = time.time()

        # 调用下一个拦截器 / 实际处理函数
        response = continuation(handler_call_details)

        elapsed = time.time() - start
        print(f"[gRPC Server] ← 完成请求: {method} ({elapsed:.3f}s)")

        return response
```

### 注册到服务器

```python
server = grpc.server(
    futures.ThreadPoolExecutor(max_workers=10),
    interceptors=[LoggingInterceptor()]          # ← 在这里注册
)
```

### 效果

启动服务端后，每次客户端调用，终端会打印：

```
[gRPC Server] → 收到请求: /user.UserService/CreateUser
[gRPC Server] ← 完成请求: /user.UserService/CreateUser (0.002s)
[gRPC Server] → 收到请求: /user.UserService/GetUser
[gRPC Server] ← 完成请求: /user.UserService/GetUser (0.001s)
```

## 四、服务端认证拦截器

### 代码

```python
class AuthInterceptor(grpc.ServerInterceptor):
    """
    认证拦截器：检查请求的 metadata 中是否包含有效的 token
    """

    def intercept_service(self, continuation, handler_call_details):
        # 1. 获取请求的 metadata（类似 HTTP Header）
        metadata = dict(handler_call_details.invocation_metadata)

        token = metadata.get('authorization', '')
        if token != 'Bearer secret-token-123':
            # 2. 认证失败：直接返回一个拒绝的响应
            def abort_handler(request, context):
                context.abort(
                    grpc.StatusCode.UNAUTHENTICATED,
                    '认证失败：无效的 token'
                )
            # 返回一个包装的处理函数
            return grpc.unary_unary_rpc_method_handler(abort_handler)

        # 3. 认证通过：正常调用后续处理
        return continuation(handler_call_details)
```

### 注册（可以同时注册多个拦截器）

```python
server = grpc.server(
    futures.ThreadPoolExecutor(max_workers=10),
    interceptors=[
        LoggingInterceptor(),    # 先记日志
        AuthInterceptor(),       # 再验认证
    ]
)
```

> 多个拦截器的执行顺序：按列表顺序依次调用，像"洋葱圈"一样层层包裹。

## 五、客户端拦截器

客户端也有拦截器，最常用的是"自动给每个请求加认证头"。

### 一元 RPC 客户端拦截器

```python
class ClientAuthInterceptor(grpc.UnaryUnaryClientInterceptor):
    """
    客户端认证拦截器：
    自动在每个请求的 metadata 中附加 token
    """

    def __init__(self, token):
        self.token = token

    def intercept_unary_unary(self, continuation, client_call_details, request):
        """
        continuation: 调用链的下一个
        client_call_details: 调用详情（方法、超时、metadata 等）
        request: 请求对象
        """
        # 1. 添加认证 metadata
        new_metadata = []
        if client_call_details.metadata:
            new_metadata = list(client_call_details.metadata)
        new_metadata.append(('authorization', f'Bearer {self.token}'))

        # 2. 创建新的 call_details（替换 metadata）
        new_details = client_call_details._replace(metadata=new_metadata)

        # 3. 继续调用链
        return continuation(new_details, request)
```

### 注册到 Channel

```python
# 普通 channel（不安全连接）
raw_channel = grpc.insecure_channel('localhost:50051')

# 包装上拦截器
channel = grpc.intercept_channel(
    raw_channel,
    ClientAuthInterceptor(token='secret-token-123')
)

# 之后使用 channel 创建的 stub，每个请求都会自动带上 token
stub = user_pb2_grpc.UserServiceStub(channel)
```

## 六、流式拦截器

流式 RPC 也有对应的拦截器：

| RPC 类型 | 客户端拦截器基类 |
|---------|----------------|
| 一元 RPC | `UnaryUnaryClientInterceptor` |
| 服务端流式 | `UnaryStreamClientInterceptor` |
| 客户端流式 | `StreamUnaryClientInterceptor` |
| 双向流式 | `StreamStreamClientInterceptor` |

模式和一元类似，这里不展开，需要时查文档即可。

## 七、完整集成示例

### 改造后的服务端（`grpc_server_v2.py`）

```python
import grpc
from concurrent import futures
import time
import user_pb2, user_pb2_grpc

# ===== 拦截器定义 =====
class LoggingInterceptor(grpc.ServerInterceptor):
    def intercept_service(self, continuation, handler_call_details):
        method = handler_call_details.method
        print(f"[{time.strftime('%H:%M:%S')}] → {method}")
        start = time.time()
        response = continuation(handler_call_details)
        print(f"[{time.strftime('%H:%M:%S')}] ← {method} ({time.time()-start:.3f}s)")
        return response

class AuthInterceptor(grpc.ServerInterceptor):
    def intercept_service(self, continuation, handler_call_details):
        md = dict(handler_call_details.invocation_metadata)
        if md.get('authorization') != 'Bearer my-token':
            def deny(request, context):
                context.abort(grpc.StatusCode.UNAUTHENTICATED, 'Token 无效')
            return grpc.unary_unary_rpc_method_handler(deny)
        return continuation(handler_call_details)

# ===== 服务实现（同第04课）=====
class UserService(user_pb2_grpc.UserServiceServicer):
    # ... 与之前相同 ...

# ===== 启动服务器（带拦截器）=====
server = grpc.server(
    futures.ThreadPoolExecutor(max_workers=10),
    interceptors=[LoggingInterceptor(), AuthInterceptor()]
)
user_pb2_grpc.add_UserServiceServicer_to_server(UserService(), server)
server.add_insecure_port('[::]:50051')
server.start()
print("🚀 gRPC 服务已启动（带日志 + 认证拦截器）")
server.wait_for_termination()
```

### 改造后的客户端（`grpc_client_v2.py`）

```python
import grpc
import user_pb2, user_pb2_grpc

class ClientAuthInterceptor(grpc.UnaryUnaryClientInterceptor):
    def __init__(self, token):
        self.token = token
    def intercept_unary_unary(self, continuation, details, request):
        md = list(details.metadata) if details.metadata else []
        md.append(('authorization', f'Bearer {self.token}'))
        return continuation(details._replace(metadata=md), request)

# 创建带拦截器的 channel
raw = grpc.insecure_channel('localhost:50051')
channel = grpc.intercept_channel(raw, ClientAuthInterceptor('my-token'))
stub = user_pb2_grpc.UserServiceStub(channel)

# 正常调用（自动带上 token）
resp = stub.CreateUser(user_pb2.CreateUserRequest(name="Alice", email="alice@e.com"))
print(f"创建成功: {resp.id}")
```

## 八、对比总结

| 概念 | FastAPI | gRPC |
|------|---------|------|
| 请求前处理 | `@app.middleware` | `ServerInterceptor.intercept_service()` |
| 请求后处理 | 同上 | 同上（在 `continuation` 返回后） |
| 认证 | Middleware 检查 Header | Interceptor 检查 metadata |
| 客户端加 Header | `client.headers = {"auth": "xxx"}` | `ClientInterceptor` 注入 metadata |
| 多个拦截器顺序 | 注册顺序 | 列表顺序（洋葱模型） |

## 九、本课小结

```
拦截器 = gRPC 的 Middleware

  服务端拦截器：ServerInterceptor
    → 日志、认证、限流、metrics 埋点

  客户端拦截器：UnaryUnaryClientInterceptor
    → 自动加 token、重试、超时

  注册方式：
    服务端：grpc.server(interceptors=[...])
    客户端：grpc.intercept_channel(channel, ...)
```
