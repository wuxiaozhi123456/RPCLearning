# 第 12 课：调试工具 —— 像 curl 和 Postman 一样调试 gRPC

> **目标**：掌握 grpcurl、grpcui、evans 三个工具，告别"写客户端代码才能测试"的尴尬  
> **对照**：grpcurl ≈ curl，grpcui ≈ Postman/Swagger UI

---

## 一、痛点：调试 gRPC 很麻烦？

REST 时代，调试一个接口很简单：

```bash
curl -X POST http://localhost:8000/users \
  -H "Content-Type: application/json" \
  -d '{"name":"Alice","email":"alice@example.com"}'
```

或者打开浏览器访问 `http://localhost:8000/docs`，Swagger UI 直接交互。

gRPC 呢？每次都要写客户端代码？

> [ERR] 不需要！gRPC 生态有对应的调试工具。

---

## 二、工具一览

| 工具 | 对应 REST 工具 | 类型 | 安装 |
|------|:---:|------|------|
| **grpcurl** | `curl` | CLI 命令行 | 需 Go 环境 |
| **grpcui** | Postman / Swagger UI | Web UI | 需 Go 环境 |
| **evans** | 交互式 REPL | CLI 交互 | 需 Go 环境 |

---

## 三、grpcurl —— 命令行调试

### 3.1 安装

```bash
# 需要先安装 Go，然后：
go install github.com/fullstorydev/grpcurl/cmd/grpcurl@latest

# 验证
grpcurl --version
```

> 如果不想装 Go，可以去 [GitHub Releases](https://github.com/fullstorydev/grpcurl/releases) 下载编译好的二进制文件。

### 3.2 列出所有服务

```bash
# 列出服务器上注册的所有 gRPC 服务
grpcurl -plaintext localhost:50051 list
```

输出：
```
user.UserService
grpc.reflection.v1alpha.ServerReflection
```

### 3.3 列出某个服务的方法

```bash
grpcurl -plaintext localhost:50051 list user.UserService
```

输出：
```
user.UserService.CreateUser
user.UserService.GetUser
user.UserService.ListUsers
```

### 3.4 查看方法签名

```bash
grpcurl -plaintext localhost:50051 describe user.UserService.CreateUser
```

输出：
```
user.UserService.CreateUser is a rpc:
rpc CreateUser ( .user.CreateUserRequest ) returns ( .user.UserResponse );
```

### 3.5 查看消息结构

```bash
grpcurl -plaintext localhost:50051 describe user.CreateUserRequest
```

输出：
```
user.CreateUserRequest is a message:
message CreateUserRequest {
  string name = 1;
  string email = 2;
}
```

### 3.6 调用方法

```bash
# 调用 CreateUser（JSON 格式传参）
grpcurl -plaintext \
  -d '{"name": "Alice", "email": "alice@example.com"}' \
  localhost:50051 \
  user.UserService/CreateUser
```

输出：
```json
{
  "id": 1,
  "name": "Alice",
  "email": "alice@example.com"
}
```

```bash
# 调用 GetUser
grpcurl -plaintext \
  -d '{"id": 1}' \
  localhost:50051 \
  user.UserService/GetUser

# 触发错误
grpcurl -plaintext \
  -d '{"id": 999}' \
  localhost:50051 \
  user.UserService/GetUser
```

### 3.7 常用选项

| 选项 | 说明 |
|------|------|
| `-plaintext` | 明文连接（无 TLS） |
| `-d '{"key":"val"}'` | 请求体（JSON） |
| `-H 'key: value'` | 添加 metadata（类似 HTTP Header） |
| `-max-msg-sz 10485760` | 最大消息大小（默认 4MB） |
| `-rpc-header 'auth: token'` | 请求头 |

## 四、grpcui —— Web UI 交互式调试

### 4.1 安装

```bash
go install github.com/fullstorydev/grpcui/cmd/grpcui@latest
```

### 4.2 启动 Web UI

```bash
grpcui -plaintext localhost:50051
```

浏览器自动打开，你会看到一个类似 Swagger UI 的界面：

```
┌──────────────────────────────────────────────┐
│  gRPC UI                                     │
│                                              │
│  📁 user.UserService                         │
│    ├── CreateUser    [Invoke]                │
│    ├── GetUser       [Invoke]                │
│    └── ListUsers     [Invoke]                │
│                                              │
│  选择一个方法 → 填写 JSON → 点击 Invoke       │
└──────────────────────────────────────────────┘
```

> 就像 FastAPI 的 Swagger UI！选择接口、填参数、点发送、看结果。

---

## 五、⚠️ 前提条件：开启 gRPC Reflection

grpcurl 和 grpcui 都依赖 **gRPC Reflection**（服务反射）。如果没有开启，它们不知道服务有哪些方法。

### 开启方法

```python
from grpc_reflection.v1alpha import reflection

# 需要安装
# pip install grpcio-reflection

import grpc
from concurrent import futures

server = grpc.server(futures.ThreadPoolExecutor(max_workers=10))

# 注册你的服务
user_pb2_grpc.add_UserServiceServicer_to_server(UserService(), server)

# ===== 开启 Reflection =====
SERVICE_NAMES = (
    user_pb2.DESCRIPTOR.services_by_name['UserService'].full_name,
    reflection.SERVICE_NAME,
)
reflection.enable_server_reflection(SERVICE_NAMES, server)
# ==========================

server.add_insecure_port('[::]:50051')
server.start()
server.wait_for_termination()
```

安装依赖：
```bash
pip install grpcio-reflection
```

## 六、evans —— 交互式 REPL

### 6.1 安装

```bash
go install github.com/ktr0731/evans@latest
```

### 6.2 使用

```bash
# 启动 REPL
evans --host localhost --port 50051 -r repl

# 在 REPL 中：
> show service          # 列出所有服务
> service UserService   # 选择一个服务
> show rpc              # 列出所有方法
> call CreateUser       # 调用方法（交互式填参数）
  name (TYPE_STRING) => Alice
  email (TYPE_STRING) => alice@example.com
  { "id": 1, "name": "Alice", "email": "alice@example.com" }
```

## 七、工具选择建议

| 场景 | 推荐工具 |
|------|---------|
| 快速验证一个接口 | `grpcurl` |
| 浏览所有接口，交互式调试 | `grpcui` |
| 喜欢命令行 REPL 风格 | `evans` |
| CI/CD 中做集成测试 | `grpcurl` + shell 脚本 |
| 给前端/QA 做接口文档 | `grpcui` 截图或分享 URL |

## 八、完整调试流程总结

```bash
# 1. 确保服务端开启了 Reflection
# 2. 看看有哪些服务
grpcurl -plaintext localhost:50051 list

# 3. 看看服务有哪些方法
grpcurl -plaintext localhost:50051 list user.UserService

# 4. 看看方法参数长什么样
grpcurl -plaintext localhost:50051 describe user.CreateUserRequest

# 5. 调用它！
grpcurl -plaintext -d '{"name":"Test"}' localhost:50051 user.UserService/CreateUser

# 6. 需要更好体验？用 Web UI
grpcui -plaintext localhost:50051
```

## 九、本课小结

```
不再需要"写客户端代码 → 运行 → 看结果"的循环！

  REST 有的调试工具        gRPC 都有对应的
  ────────────────        ────────────────
  curl                →   grpcurl
  Postman             →   grpcui
  Swagger UI          →   grpcui（自动发现方法）
  httpie / 交互式     →   evans

  前提：开启 gRPC Reflection（pip install grpcio-reflection）
```
