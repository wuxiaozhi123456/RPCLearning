# 第 07 课：错误处理 —— gRPC 如何表示"出错了"

> **目标**：学会在服务端返回 gRPC 状态码，在客户端捕获并处理错误  
> **对照**：理解 gRPC StatusCode 和 HTTP 状态码的对应关系

---

## 一、REST 的错误处理（你已熟悉）

```python
# FastAPI 服务端
raise HTTPException(status_code=404, detail="用户不存在")

# FastAPI 客户端
if resp.status_code == 404:
    print("用户不存在")
```

很简单：**服务端设置 HTTP 状态码 → 客户端检查状态码**。

---

## 二、gRPC 的错误处理：两种方式

### 方式 1：`context.set_code()` + `context.set_details()`（推荐）

```python
def GetUser(self, request, context):
    if request.id not in self.db:
        context.set_code(grpc.StatusCode.NOT_FOUND)
        context.set_details(f"用户 {request.id} 不存在")
        return user_pb2.UserResponse()  # 必须返回一个消息
    return self.db[request.id]
```

### 方式 2：`context.abort()`（更简洁，直接终止）

```python
def GetUser(self, request, context):
    if request.id not in self.db:
        context.abort(grpc.StatusCode.NOT_FOUND, f"用户 {request.id} 不存在")
        # abort() 会抛异常，下面的代码不会执行
    return self.db[request.id]
```

| | `set_code()` | `abort()` |
|------|:---:|:---:|
| 是否终止执行 | ❌ 继续执行 | ✅ 直接抛出异常 |
| 是否必须 return | ✅ 必须手动 return | ❌ 不需要 |
| 适用场景 | 需要做额外清理再返回 | 快速失败 |

## 三、完整服务端示例（改造 GetUser）

```python
def GetUser(self, request, context):
    user_id = request.id

    # 参数校验
    if user_id <= 0:
        context.abort(
            grpc.StatusCode.INVALID_ARGUMENT,
            f"用户 ID 必须为正数，收到: {user_id}"
        )

    # 业务逻辑：未找到
    if user_id not in self.db:
        context.abort(
            grpc.StatusCode.NOT_FOUND,
            f"用户 {user_id} 不存在"
        )

    # 正常返回
    return self.db[user_id]
```

## 四、客户端捕获错误

```python
import grpc

# 调用可能出错的方法
try:
    resp = stub.GetUser(
        user_pb2.GetUserRequest(id=-1),
        timeout=2.0
    )
except grpc.RpcError as e:
    code = e.code()       # grpc.StatusCode 枚举
    details = e.details() # 错误详情字符串
    
    print(f"❌ 调用失败!")
    print(f"   状态码: {code.name} ({code.value[0]})")
    print(f"   详情:   {details}")
    
    # 根据不同的错误类型做不同处理
    if code == grpc.StatusCode.NOT_FOUND:
        print("   → 用户不存在，返回 404 给前端")
    elif code == grpc.StatusCode.INVALID_ARGUMENT:
        print("   → 参数错误，提示用户")
    elif code == grpc.StatusCode.UNAVAILABLE:
        print("   → 服务不可用，触发告警")
```

## 五、gRPC 常用状态码速查

| gRPC StatusCode | 含义 | 对应 HTTP | 使用场景 |
|----------------|------|:---:|------|
| `OK` | 成功 | 200 | 正常返回 |
| `NOT_FOUND` | 资源不存在 | 404 | GetUser(id=999) |
| `INVALID_ARGUMENT` | 参数错误 | 400 | name 为空 |
| `ALREADY_EXISTS` | 资源已存在 | 409 | 重复注册 |
| `PERMISSION_DENIED` | 无权限 | 403 | 非管理员 |
| `UNAUTHENTICATED` | 未认证 | 401 | 缺少 token |
| `INTERNAL` | 服务器内部错误 | 500 | 数据库挂了 |
| `UNAVAILABLE` | 服务不可用 | 503 | 网络不通 |
| `DEADLINE_EXCEEDED` | 超时 | 504 | 请求处理太久 |
| `UNIMPLEMENTED` | 方法未实现 | 501 | 接口还没写 |

## 六、实战：改造你的服务端

把 `grpc_server.py` 的 `GetUser` 改成完整版本：

```python
def GetUser(self, request, context):
    user_id = request.id
    
    # 1. 参数校验
    if not isinstance(user_id, int) or user_id <= 0:
        context.abort(
            grpc.StatusCode.INVALID_ARGUMENT,
            f"Invalid user_id: {user_id}. Must be a positive integer."
        )
    
    # 2. 查数据
    if user_id not in self.db:
        context.abort(
            grpc.StatusCode.NOT_FOUND,
            f"User with ID {user_id} not found"
        )
    
    # 3. 正常返回
    return self.db[user_id]


def CreateUser(self, request, context):
    # 参数校验
    if not request.name.strip():
        context.abort(
            grpc.StatusCode.INVALID_ARGUMENT,
            "Name cannot be empty"
        )
    
    if "@" not in request.email:
        context.abort(
            grpc.StatusCode.INVALID_ARGUMENT,
            "Invalid email format"
        )
    
    # 检查重复
    for user in self.db.values():
        if user.email == request.email:
            context.abort(
                grpc.StatusCode.ALREADY_EXISTS,
                f"Email '{request.email}' already registered"
            )
    
    # 正常创建
    user = user_pb2.UserResponse(
        id=self.next_id,
        name=request.name,
        email=request.email
    )
    self.db[self.next_id] = user
    self.next_id += 1
    return user
```

## 七、元数据（Metadata）—— 类似 HTTP Header

除了状态码，你还可以在响应中附加元数据（类似 HTTP 的 Header）：

**服务端设置**：
```python
def GetUser(self, request, context):
    # 添加响应的 metadata（类似 HTTP Response Header）
    trailing_metadata = [('x-request-cost', '23ms')]
    context.set_trailing_metadata(trailing_metadata)
    return self.db[request.id]
```

**客户端读取**：
```python
try:
    resp = stub.GetUser(request)
    # 注意：trailing metadata 在 for 循环后可用
except grpc.RpcError as e:
    # 从异常中获取 trailing metadata
    print(f"Metadata: {e.trailing_metadata()}")
```

## 八、本课小结

```
gRPC 错误处理的模式：

  服务端                      客户端
  ───────                     ───────
  context.abort(CODE, msg)    try:
  或                            stub.Method(req)
  context.set_code(CODE)      except grpc.RpcError as e:
  context.set_details(msg)        e.code()     ← 错误类型
                                  e.details()  ← 错误详情
  
  和 REST 的对照：
  NOT_FOUND        ←→  404
  INVALID_ARGUMENT ←→  400
  INTERNAL         ←→  500
  ALREADY_EXISTS   ←→  409
  UNAVAILABLE      ←→  503
```
